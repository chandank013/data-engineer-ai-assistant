Connection

Spark / Hadoop (.env)
# Paste into your .env file SPARK_MASTER=local[*] SPARK_DRIVER_MEMORY=2g SPARK_SHUFFLE_PARTITIONS=4 # local[*] = all CPU cores | local[2] = limit to 2 cores
✅ No cluster needed — runs on your machine
✅ Files uploaded through the Datasets tab (local disk)
✅ Best for development, testing and datasets under 10 GB
# Paste into your .env file SPARK_MASTER=yarn HDFS_NAMENODE=hdfs://namenode:9000 YARN_RESOURCE_MANAGER=resourcemanager:8032 SPARK_EXECUTOR_MEMORY=4g SPARK_EXECUTOR_CORES=2 SPARK_NUM_EXECUTORS=4 SPARK_DRIVER_MEMORY=4g
⚠️ Also set HADOOP_CONF_DIR to your cluster's config folder
⚠️ Your machine must be able to reach the namenode over the network
✅ Scales to TB-size datasets across many nodes
# Paste into your .env file SPARK_MASTER=spark://master-host:7077 SPARK_EXECUTOR_MEMORY=4g SPARK_EXECUTOR_CORES=2 SPARK_NUM_EXECUTORS=4 SPARK_DRIVER_MEMORY=4g
⚠️ Requires a running Spark master at spark://master-host:7077
✅ Good for dedicated Spark clusters without Hadoop/YARN

HDFS Status & Files

Shows the HDFS filesystem status and files in /data/. In local mode this will show "not configured" — that is normal. In Hadoop YARN mode it lists every file synced to HDFS so you can confirm uploads reached the cluster. If you see "hdfs command not found" it means Hadoop is not yet installed — follow the WSL2 setup guide in the switch guide below.

Data Flow: Browser → HDFS → Spark
[ Browser ] → Upload CSV/XLSX/JSON via Datasets tab ↓ [ Flask ] → Saves to data/raw/filename.csv (local disk) ↓ (if HDFS_NAMENODE set in .env) [ HDFS ] → Auto-syncs to hdfs://namenode:9000/data/raw/filename.csv ↓ [ Spark ] → Reads from HDFS (YARN mode) or local (local[*] mode) ↓ [ Flask ] → Returns results to browser ✅
How auto-sync works: Every time you upload a CSV, XLSX or JSON file through the Datasets tab, Flask automatically runs hdfs dfs -put -f file /data/raw/file in the background — only when HDFS_NAMENODE is set in your .env. In local mode the HDFS step is skipped silently.
